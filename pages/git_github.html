Gestión de Proyectos con Git y GitHub

1. Configurar Git en tu ordenador

Primero, asegúrate de tener Git instalado. Abre la terminal y configura tu nombre y correo electrónico:

git config --global user.name "Tu Nombre"
git config --global user.email "tu_email@example.com"

2. Crear un repositorio local

Crea un nuevo proyecto o navega a la carpeta donde ya tienes un proyecto. Luego, inicializa un repositorio de Git:

git init

3. Hacer el primer commit

  1. Añade los archivos que deseas gestionar a Git: 
    git add .
  2. Guarda los cambios en el repositorio local con un "commit": 
    git commit -m "Mensaje inicial del proyecto"

4. Crear un repositorio en GitHub

Ve a GitHub y haz clic en New repository. Dale un nombre al repositorio y configura las opciones. Cópiala URL proporcionada para conectar tu repositorio local con GitHub.

5. Conectar el repositorio local con GitHub

git remote add origin https://github.com/tu_usuario/tu_repositorio.git

6. Subir cambios a GitHub

Para subir tu proyecto por primera vez, usa el siguiente comando:

git push -u origin main

7. Hacer cambios y gestionarlos con Git

  1. Añadir cambios: git add .
  2. Guardar cambios: git commit -m "Descripción de los cambios"
  3. Subir cambios: git push origin main

8. Colaborar con otros

Comandos útiles para colaborar en equipo:

  • Clonar un repositorio: 
    git clone https://github.com/usuario/repositorio.git
  • Crear ramas: 
    git checkout -b nombre_de_rama
  • Fusionar ramas: 
    git checkout main
git merge nombre_de_rama
  • Actualizar el repositorio local: git pull origin main

9. Usar Pull Requests en GitHub

  1. Sube la rama con git push origin nombre_de_rama.
  2. En GitHub, abre un Pull Request para solicitar la revisión e integración de tus cambios.

10. Resolver conflictos

Si hay conflictos al fusionar ramas, Git te lo notificará. Edita los archivos en conflicto y realiza un commit para confirmar la resolución.

11. Ejemplo de flujo de trabajo (workflow)

Un flujo de trabajo común incluye:

  1. Clonar el repositorio: git clone https://github.com/usuario/proyecto.git
  2. Crear una nueva rama para cada nueva característica: git checkout -b nueva_caracteristica
  3. Hacer commits mientras trabajas en la nueva característica.
  4. Subir la rama a GitHub: git push origin nueva_caracteristica
  5. Abrir un Pull Request en GitHub.
  6. Fusionar el Pull Request en GitHub.
  7. Actualizar la rama principal localmente: 
    git checkout main
git pull origin main
